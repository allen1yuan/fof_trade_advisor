# 基于基金画像因子与机器学习的 FOF 量化选基策略研究

## 项目简介

本项目聚焦 FOF（Fund of Funds）量化选基问题，尝试构建一套基于 **基金画像因子 + 机器学习模型 + 回测验证** 的基金筛选框架。

项目核心目标是：  
通过刻画基金的收益表现、风险控制、风险调整收益、基金经理能力、持仓风格、行业配置、规模变化与业绩持续性等多维特征，构建基金画像因子体系，并结合机器学习方法对基金未来表现进行预测、排序和组合构建，最终形成一套 **可解释、可回测、可复现、可迭代优化** 的 FOF 量化选基策略。

第一阶段研究对象主要聚焦于 **主动管理型偏股基金**，后续可扩展至债券型基金、“固收+”基金、QDII 基金等其他 FOF 子策略。

---

## 研究背景

随着公募基金数量持续增长，传统依赖历史收益排名或基金评级的选基方式存在一定局限：

- 历史收益具有滞后性，难以直接代表未来表现
- 短期排名容易出现均值回归
- 高收益可能只是来自高 Beta 或特定市场风格暴露
- 基金经理变更、规模变化、风格漂移等因素容易被忽略
- 主观选基难以形成稳定、可复现的研究流程

因此，本项目希望通过数据化方式构建基金画像因子，并结合机器学习方法识别影响基金未来表现的关键特征，为 FOF 组合构建提供量化依据。

---

## 核心研究问题

本项目重点关注以下问题：

1. 如何构建多维度、可解释的基金画像因子体系？
2. 哪些因子对基金未来收益和风险调整收益具有预测能力？
3. 机器学习模型能否提升 FOF 选基效果？
4. 如何将模型预测结果转化为可执行的基金组合？
5. 如何通过回测验证策略的稳定性、有效性和可泛化能力？

## 基金池定义

第一阶段建议聚焦 **主动管理型偏股基金**，包括普通股票型基金、偏股混合型基金，以及权益仓位较高的灵活配置型基金。

| 筛选维度 | 筛选标准 | 目的 |
|---|---|---|
| 成立时间 | 成立满 2 年以上 | 保证历史样本充足 |
| 基金规模 | 例如 2 亿—100 亿 | 避免迷你基金清盘风险与超大规模灵活度不足 |
| 基金经理 | 过去 1 年内未发生核心经理变更 | 保证历史业绩与现任经理能力相关 |
| 基金类型 | 主动权益 / 偏股混合为主 | 聚焦主动选基能力 |
| 数据完整性 | 净值、规模、持仓等关键字段缺失较少 | 保证因子计算质量 |
| 流动性 | 剔除申赎限制较强的产品 | 保证策略可执行性 |

## 数据需求与数据契约

### 数据需求

| 数据类型 | 主要字段 | 用途 |
|---|---|---|
| 基金基本信息 | 基金代码、名称、类型、成立日期、基金公司 | 构建基金池 |
| 基金净值数据 | 复权净值、日收益率 | 计算收益、风险和回撤因子 |
| 基金持仓数据 | 股票持仓、债券持仓、行业分布、前十大重仓 | 构建持仓、行业和风格因子 |
| 基金经理数据 | 任职时间、历史管理基金、任职期收益 | 构建基金经理能力因子 |
| 基金规模数据 | 资产净值、规模变化率 | 分析规模效应 |
| 持有人结构 | 机构持有人比例、个人持有人比例 | 衡量资金结构与认可度 |
| 市场指数数据 | 沪深 300、中证 500、中证偏股基金指数等 | 计算超额收益、Beta 和跟踪误差 |
| 风格因子数据 | 价值、成长、动量、质量、波动率等 | 拆解基金风格暴露 |

### 标准数据表设计

| 表名 | 索引 | 主要字段 |
|---|---|---|
| `nav_df` | `(date, fund_id)` | `adj_nav`, `ret` |
| `holding_df` | `(date, fund_id, stock_id)` | `weight`, `industry`, `mkt_value` |
| `bench_df` | `(date, bench_id)` | `ret` |
| `factor_ret_df` | `(date, factor_name)` | `ret` |
| `fund_info_df` | `fund_id` | `type`, `setup_date`, `manager_id`, `manager_start_date`, `company` |

### 数据质量要求

- 使用复权净值，避免分红对收益计算造成干扰
- 注意基金季报、半年报和年报披露存在时间滞后
- 避免未来函数，所有持仓和基金经理信息必须按真实可得时间处理
- 尽量保留历史清盘基金，降低幸存者偏差
- 对净值异常跳变、缺失数据和极端值进行检测

## 基金画像因子体系

本项目将基金画像因子分为 8 大类。

| 因子类别 | 代表指标 | 研究目的 |
|---|---|---|
| 业绩表现因子 | 近 3 月、6 月、12 月、24 月收益率，超额收益 | 衡量基金历史收益能力 |
| 风险特征因子 | 年化波动率、最大回撤、下行波动率、VaR、CVaR | 衡量基金风险暴露 |
| 风险调整收益因子 | Sharpe、Sortino、Calmar、Information Ratio | 衡量单位风险下的收益质量 |
| 风格特征因子 | 大小盘、价值、成长、动量、质量、波动率暴露 | 拆解基金收益来源 |
| 行业配置因子 | 行业集中度、行业偏离度、前三大行业权重 | 衡量行业配置风险 |
| 持仓特征因子 | 前十大持仓占比、个股集中度、换手率、抱团暴露 | 判断持仓结构与交易特征 |
| 基金经理因子 | 任职年限、任职期超额收益、T-M Alpha、H-M Alpha | 衡量基金经理管理能力 |
| 业绩持续性因子 | 月度胜率、季度胜率、滚动 IR、排名稳定性 | 判断基金表现是否持续稳定 |

### 1. 业绩表现因子

## 因子预处理

为了保证不同因子之间可比较，本项目将对因子进行统一预处理。

```text
原始因子
   ↓
缺失值处理
   ↓
异常值处理
   ↓
标准化
   ↓
中性化
   ↓
方向统一
   ↓
因子合成 / 模型输入
```

| 步骤 | 方法 | 目的 |
|---|---|---|
| 缺失值处理 | 中位数填充、分组填充、剔除严重缺失因子 | 保证样本完整性 |
| 异常值处理 | Winsorize、MAD 缩尾 | 降低极端值影响 |
| 标准化 | Z-score、Rank 标准化 | 统一因子量纲 |
| 中性化 | 对基金类型、规模、成立时间做回归取残差 | 剥离非核心影响 |
| 方向统一 | 对负向因子取反 | 保证因子值越高代表越优 |

## 因子有效性检验

每个因子在进入模型前，需要先进行有效性检验。

| 检验方法 | 说明 | 目的 |
|---|---|---|
| IC 检验 | 计算因子值与未来收益之间的相关性 | 判断因子预测能力 |
| Rank IC 检验 | 使用秩相关衡量排序能力 | 更适合选基排序场景 |
| IC_IR | 因子 IC 均值 / IC 标准差 | 衡量因子稳定性 |
| 分组回测 | 将基金按因子值分组，比较不同组收益 | 检验因子单调性 |
| 相关性筛查 | 剔除高度相关的冗余因子 | 降低信号重复 |
| 市场分段检验 | 在牛市、熊市、震荡市分别检验 | 判断因子稳健性 |

| 标签类型 | 定义 | 适用任务 |
|---|---|---|

| 未来风险调整收益 | 未来 Sharpe / Calmar / IR | 回归或排序 |

## 机器学习建模思路

本项目将 FOF 选基问题视为 **预测 + 排序** 问题。

### 1. 基准模型

| 模型 | 用途 |
|---|---|
| 单因子排序 | 检验单个因子的有效性 |
| 多因子等权打分 | 作为最简单的基准策略 |
| IC 加权打分 | 根据历史因子有效性动态分配权重 |
| IC_IR 加权打分 | 同时考虑因子有效性和稳定性 |

### 2. 机器学习模型

| 模型 | 任务 | 说明 |
|---|---|---|
| Ridge / Lasso | 回归 | 可解释性强，适合作为线性基准 |
| Random Forest | 回归 / 分类 | 捕捉非线性关系 |
| XGBoost | 回归 / 分类 / 排序 | 适合结构化因子数据 |
| LightGBM | 回归 / 分类 / 排序 | 训练效率高，适合横截面排序 |
| LightGBM Ranker | 排序 | 更贴近 Top-K 选基场景 |

### 3. 模型解释

为了提高策略可信度，本项目将引入模型解释方法：

- 特征重要性分析
- SHAP Value
- 因子分组回测
- 模型预测分数分层检验
- 不同市场环境下的因子贡献分析

## 因子合成与基金评分

本项目将采用“两层合成”的方式构建基金综合评分。

### 第一层：大类因子内部合成

先在每一类因子内部合成子评分，例如收益得分、风险得分、经理能力得分等。

可选方法：

- 等权合成
- IC 加权合成
- IC_IR 加权合成

### 第二层：跨类别综合评分

将各大类因子得分合成为最终基金评分。

```text
Fund Score =
w1 × 收益表现得分
+ w2 × 风险控制得分
+ w3 × 风险调整收益得分
+ w4 × 基金经理得分
+ w5 × 持仓风格得分
+ w6 × 业绩持续性得分
```

综合评分可用于：

- 基金排序
- Top-K 基金筛选
- 组合权重分配
- 模型预测结果解释

## 选基策略构建流程

1. **确定基金池**  
   剔除成立时间过短、规模过小、数据缺失严重或流动性受限的基金。

2. **构建基金画像因子**  
   从收益、风险、稳定性、基金经理、持仓风格、行业配置等维度构建特征。

3. **进行因子预处理和有效性检验**  
   对因子进行标准化、中性化、IC 检验和分组回测。

4. **训练机器学习模型**  
   使用历史样本训练模型，预测基金未来收益、超额收益或排名。

5. **生成基金评分与排序**  
   根据模型输出结果对基金进行打分。

6. **构建 FOF 组合**  
   选择排名靠前的基金构建 Top-K 组合。

7. **定期调仓与风险控制**  
   按月度或季度更新因子、模型评分和组合权重。

8. **回测与绩效评估**  
   验证策略是否具备稳定超额收益和可控回撤。

---

## 组合构建方法

| 方法 | 说明 | 适用场景 |
|---|---|---|
| 等权组合 | Top-K 基金平均配置 | 基准组合 |
| 模型分数加权 | 模型评分越高，权重越高 | 强调预测能力 |
| 波动率倒数加权 | 波动率越低，权重越高 | 风险控制优先 |
| 风险平价 | 每只基金贡献相近风险 | 风险均衡组合 |
| 均值-方差优化 | 在风险约束下优化收益 | 进阶组合优化 |

### 组合约束

- 单只基金权重上限：例如 20%
- 单一基金经理权重上限：例如 30%
- 单一基金公司权重上限：例如 40%
- 持仓基金数量：例如 5—15 只
- 控制行业、风格和资产类别集中度
- 控制组合整体波动率和最大回撤

## 风险管理

### 1. 基金层面风险控制

- 剔除规模过小或规模变化异常的基金
- 剔除成立时间过短的基金
- 剔除基金经理近期变更的基金
- 剔除历史最大回撤过大的基金
- 剔除数据质量较差的基金

### 2. 组合层面风险控制

- 限制单只基金最大权重
- 控制同一基金经理和基金公司集中度
- 控制行业和风格暴露
- 控制组合整体波动率和最大回撤
- 控制与基准的偏离程度

### 3. 模型层面风险控制

- 避免未来函数和数据泄露
- 使用时间序列样本外测试
- 控制模型复杂度
- 对因子稳定性进行滚动检验
- 避免过度依赖单一模型结果

### 4. 策略监控机制

| 预警信号 | 触发条件 | 应对方式 |
|---|---|---|
| 组合大幅回撤 | 回撤超过预设阈值 | 降低权益暴露或重新评估持仓 |
| 单基金异常下跌 | 单月显著跑输基准 | 触发人工复核 |
| 基金经理变更 | 持仓基金经理发生变化 | 重新计算经理因子并列入观察 |
| 规模异常变化 | 规模短期大幅上升或下降 | 降权或剔除 |
| 因子失效 | IC 持续下降或符号反转 | 降低因子权重或移除 |

## 答辩介绍

本项目研究的是如何利用基金画像因子和机器学习方法构建 FOF 量化选基策略。首先，我会聚焦主动管理型偏股基金，构建包含收益、风险、回撤、风险调整收益、基金经理能力、持仓风格、行业配置和业绩持续性等维度的基金画像因子。随后，我会对因子进行标准化、中性化、IC 检验和分组回测，筛选出具有预测能力的有效因子。在建模阶段，我会使用多因子打分、回归模型和排序模型预测基金未来表现，并根据模型评分选择排名靠前的基金构建 FOF 组合。最后，通过样本外回测评估策略的年化收益、超额收益、最大回撤、夏普比率和信息比率，验证策略是否具备稳定性和研究价值。
